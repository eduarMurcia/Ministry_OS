# Glosario — Ministry OS

- **ADR (Architecture Decision Record):** decisión de arquitectura registrada formalmente, con justificación y consecuencias.
- **ARC-001:** Modelo Conceptual de Ministry OS. Pendiente de redactar. Definirá roles, capacidades, procesos, recursos, dominios.
- **ChMS (Church Management Software):** categoría de software (Planning Center, Breeze/Tithely) usada como referencia de mercado para no reinventar capacidades ya validadas.
- **Capacidad:** lo que la iglesia necesita poder hacer, estable e independiente de la herramienta (estilo Business Capability Map de TOGAF).
- **Recurso:** término que reemplaza "herramienta"; se clasifica en humano, documental, digital, tecnológico, inteligente.
- **Knowledge Object:** concepto propuesto para tratar todo (documento, skill, workflow, ADR) bajo una misma estructura de metadata.
- **Walking skeleton:** técnica de ingeniería recomendada para este proyecto — construir un caso real de punta a punta antes de expandir el modelo teórico completo, para evitar sobre-diseño sin validar.
